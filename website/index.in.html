GitSyncMarks

Effortless Bookmark Sync via GitHub

Your bookmarks live in your Git repo. Per-file storage, three-way merge, no middleman — syncs directly via the GitHub API.

A browser extension that bidirectionally syncs your bookmarks with a GitHub repository. Works with Chrome, Chromium, Brave, Edge, and Firefox. No third-party server — your data stays between your browser and your repo.

Core Features

No middleman

Communicates directly with the GitHub API. No third-party server, no backend. Your data stays between your browser and your GitHub repo.

Per-file storage

Each bookmark is stored as an individual JSON file in your Git repo — human-readable, diff-friendly, and easy to edit or version on GitHub.

Three-way merge

Automatic conflict-free sync when changes happen on both sides simultaneously. No manual merge needed in most cases.

Cross-browser

Works with Chrome, Chromium, Brave, Edge, and Firefox. Same extension logic, same bookmark format across all.

Auto-sync

Syncs automatically on every bookmark change. Configurable debounce so rapid edits are bundled into one sync.

Power Features

  • Multiple profiles — Up to 10 profiles (work, personal, projects). Each has its own GitHub repo config and bookmark set. Switch replaces local bookmarks with the target profile.
  • Context menu — Right-click on any page or link: Add to Toolbar, Add to Other Bookmarks, Sync Now, Switch Profile, Copy Favicon URL, Download Favicon. Bookmark auto-syncs after adding.
  • Automation — Add bookmarks via Git, CLI, or GitHub Actions. The extension picks them up automatically on the next sync.
  • GitHub Repos folder — Optional folder with bookmarks to all your GitHub repositories (public and private). Configurable position (toolbar/other).

Generated Files

GitSyncMarks can generate these files in your repo on each sync (each configurable as Off, Manual, or Auto):

  • README.md — Markdown overview of all bookmarks; browse directly on GitHub
  • bookmarks.html — Netscape format; import into Chrome, Firefox, or Edge
  • feed.xml — RSS 2.0 feed; subscribe in any reader or use for automations
  • dashy-conf.yml — Config for the Dashy dashboard

Sync Configuration

Flexible sync behavior to match your workflow:

  • Sync profiles — Real-time (1 min), Frequent (5 min), Normal (15 min), Power save (60 min), or Custom
  • Sync on startup / focus — Optional sync when the browser starts or gains focus (with cooldown)
  • Manual sync — Push, Pull, or full Sync via popup buttons
  • Conflict detection — Notifies when automatic merge is not possible; choose Push or Pull

How It Works

  1. Create a GitHub repository for your bookmarks
  2. Generate a Personal Access Token with the repo scope
  3. Install the extension and configure token, repo owner, repo name, branch, and file path
  4. Click Sync Now — your bookmarks are pushed as individual JSON files

Installation

Chrome / Chromium / Edge / Brave

  1. Go to the Releases page
  2. Download GitSyncMarks-vX.X.X-chrome.zip
  3. Extract the ZIP to a folder
  4. Open chrome://extensions/, enable Developer mode
  5. Click Load unpacked and select the extracted folder

Or install from the Chrome Web Store.

Firefox

  1. Go to the Releases page
  2. Download GitSyncMarks-vX.X.X-firefox.zip
  3. Open about:debugging#/runtime/this-firefox
  4. Click Load Temporary Add-on and select the ZIP

For a permanent install, use the Firefox Add-on store.

Repository Structure

After the first sync, your repo contains:

bookmarks/
  _index.json           # Metadata
  README.md             # Auto-generated overview
  bookmarks.html        # Netscape import
  feed.xml              # RSS 2.0 feed
  toolbar/              # Bookmarks Bar
    _order.json
    github_a1b2.json    # One file per bookmark
  other/                # Other Bookmarks
    _order.json
    ...

Each bookmark is a simple JSON file: {"title": "GitHub", "url": "https://github.com"}

Automation

Add bookmarks without opening the browser:

1. Create a file in the repo

Add a JSON file to bookmarks/toolbar/ or bookmarks/other/:

{"title": "Example", "url": "https://example.com"}

2. GitHub Action

Copy add-bookmark.yml to your repo and run:

gh workflow run add-bookmark.yml -f url="https://example.com" -f title="Example" -f folder="toolbar"

Troubleshooting

Token invalid
Ensure the PAT has the repo scope; token may have been revoked.
Repo not found
Verify Repository Owner and Name; check repo exists and you have access.
Sync takes long
Many changed bookmarks = many API calls. Enable Debug Log in Options → Sync for diagnostics.
Conflict
Choose Push or Pull in the popup when automatic merge is not possible.
