A Personal Access Token (PAT) lets GitSyncMarks access your GitHub repository to sync bookmarks. You create it in your GitHub account settings.

The token needs the repo scope. This gives read/write access to your repositories (including private ones) and lets the extension create a repo if you choose auto-create. Treat the token like a password and keep it private.

  1. Click the link below to open GitHub's token creation page.
  2. Give the token a name (e.g. "GitSyncMarks") and set an expiration if you prefer.
  3. Under "Scopes", check repo (full control of private repositories).
  4. Click "Generate token", copy it, and paste it in the field below.

    Add Bookmarks via Git

    Add bookmarks directly in your repository. The extension picks them up on the next sync and normalizes filenames automatically.

    Create file in repo

    { "title": "Example", "url": "https://example.com" }

    Folders: toolbar other

    GitHub Action

    Copy add-bookmark.yml to your repo · Raw 

    gh workflow run add-bookmark.yml \
  -f url="https://example.com" \
  -f title="Example" \
  -f folder="toolbar" \
  -f path="dev-tools" \
  -f base-path="bookmarks"

    Example: path="dev-tools" → bookmarks/toolbar/dev-tools/

    url Bookmark URL (required)
    title Title (optional; URL used if empty)
    folder Section: toolbar or other
    path Folder within section (e.g. dev-tools)
    base-path Base folder in repo (default bookmarks; must match File Path setting)

    CLI / Git workflow

    Export with the Bitwarden CLI (Vaultwarden-compatible), then commit to the same folder or upload here.

    Getting Started

    Create a GitHub repository for your bookmarks, then create a Personal Access Token (PAT) with the repo scope. In the GitHub tab, enter token, repository owner, name, branch (usually main), and the folder path (default: bookmarks). Click Test Connection to verify.

    Transfer and mirror help.

    Clean remote orphans

    On the Sync sub-tab, preview and remove remote bookmark files that are not in your local bookmarks (active profile). Useful after duplicate folders from a merge transfer.

    If the folder is empty, you can create the base structure (toolbar, other). If bookmarks already exist in the repo, you can pull them into this browser.

    In the setup wizard, Check connection only validates access — it does not sync. You then choose pull, merge/sync, push, create folder structure, or skip, with a confirmation before anything is written to the repository.

    Why does sync sometimes take long?

    GitSyncMarks uses GitHub's Git Data API for atomic commits. Each changed or new bookmark file requires an API call. With many changes, this adds up.

    All changes are written in a single atomic commit: either everything succeeds or nothing does. This prevents partial or corrupted state if the sync is interrupted. One commit instead of many also avoids hitting API rate limits and keeps the Git history clean.

    Profiles

    Each profile has its own bookmark set and GitHub repo config. Use profiles for work/personal or different devices.

    Add, Rename, Delete
    Select a profile in the dropdown, then Add (new profile), Rename, or Delete. Up to 10 profiles. Deleting removes the selected profile.
    Profile switch
    Select another profile to switch. Your current bookmarks are saved and pushed to the current profile's repo before switching. The target profile's bookmarks replace the local ones. Use "Switch without confirmation" to skip the confirm dialog.

    Profiles work reliably. With multiple profiles, sync conflicts can occasionally occur — trying Push or Pull 2–3 times usually resolves them. This feature is still evolving. If you encounter reproducible bugs, please open an issue on GitHub.

    Sync History & Undo

    Every sync creates a Git commit. You can browse recent commits in the History sub-tab (under Files) and restore bookmarks from any previous version.

    Load history
    Fetches the last 20 commits that touched your bookmark path. Each row shows the date, commit message, and a short SHA.
    Restore
    Replaces all local bookmarks with the state from that commit. This does not rewrite the remote branch — your current remote data remains untouched. The next sync will push local state back.
    Undo last sync
    One-click rollback to the commit before the most recent sync applied remote changes. Available after any pull or merge that modified local bookmarks.

    GitHub Repos Folder

    Optional: Create a folder with bookmarks to all your GitHub repositories (public and private). Choose the position (toolbar, other). The folder is synced like normal bookmarks; use Update GitHub Repos to refresh the list.

    Sync
    Auto-sync
    When enabled, syncs automatically when you add, edit, or remove bookmarks. A debounce delay waits for rapid changes to finish before syncing.
    Sync profiles
    Real-time (1 min), Frequent (5 min), Normal (15 min), or Power save (60 min). Custom lets you set interval and debounce manually.
    Sync on startup / focus
    Optional: sync when the browser starts or when you return to the browser window (with cooldown).
    Debug Log
    Enable in the Sync tab to record detailed sync diagnostics — diffs, merge decisions, and commit hashes. Export the log to share for troubleshooting.

    Menu
    Context menu tools
    In Files → Settings, you can pin up to 3 quick target folders for one-click Add actions and configure the Open All confirmation threshold. The context menu also includes Search Bookmarks, which opens a dedicated search popup.

    Linkwarden
    Integration
    Connect to your Linkwarden instance to save bookmarks directly from the context menu. You can specify default tags and auto-capture screenshots.
    Sync Collections
    Sync your Linkwarden collections into your browser bookmark folders and optionally push them to your Git repository.

    Files
    Generated files
    README.md, bookmarks.html, feed.xml, and dashy-conf.yml can be created during sync. Choose Off, Manual, or Auto per file in the Files tab.
    Settings sync to Git
    When enabled, extension settings are encrypted and stored in the repo as profile folders (profiles/<alias>/settings.enc). Import and sync explicit client profiles from the repository list.
    Export / Import
    Export bookmarks (JSON), Dashy config (YAML), or settings (plain JSON / encrypted .enc). Import bookmarks or settings — encrypted files are decrypted with your password. Bookmark import replaces all local bookmarks; settings import replaces all profiles and reloads the page.
    Bitwarden backup to Git
    Export your vault from Bitwarden or Vaultwarden as password-protected encrypted JSON, then upload it in Files → Bitwarden Backup or commit under backups/bitwarden/. List remote backups and download or delete them from the table. Optional additional Git encryption uses a local password. GitSyncMarks does not connect to Bitwarden or decrypt vault contents.
    CLI example
    Use the Bitwarden CLI: bw export --format encrypted_json --password '…' — then git add backups/bitwarden/ and commit.

    Notifications

    Choose when to show sync notifications: All (success and failure), Errors only, or Off.

    Conflicts

    When both local and remote bookmarks changed, the popup shows a conflict. Choose Push (local wins, overwrite remote) or Pull (remote wins, overwrite local). There is no automatic merge for conflicts.

    Keyboard Shortcuts
    Ctrl+Shift+. Quick sync bookmarks
    Ctrl+Shift+, Open extension settings

    Theme & Auto-save
    Language
    The menu shows short codes (e.g. EN, DE). Hover an option for the full language name.
    Theme
    Use the three icons in the header: Auto (system), Dark, or Light. The active choice is highlighted like the density buttons.
    UI Density
    Use the S / M / L buttons in the header to switch between Compact, Medium, and Large density. The setting applies to all extension surfaces and syncs across devices.
    Auto-save
    All settings save automatically when you change them — there are no Save buttons. Changes take effect immediately.
